SailAway Project
Design Dictionary

 .
Term Definition Source Date
User This object represents details about any of the User actors the application must know in order to deliver secured processes. These details include infromation like names, addresses, phone numbers, and other contact data. adiffer 2002/02/25
Library This object represents details about the Library actor the application must know in order to transact with remote libraries. At a minimum, details about the local library will be included so the application may store objects in the local file space. adiffer 2002/02/25
Notification Tool This object represents details about the Notification Tool actor the application must know in order to deliver outbound messages from the application to the User. At a minimum, details concerning the sending of email will be included so the application may send messages to the user's designated mail box. adiffer 2002/02/25
Simulation Clock This object represents details about the Simulation Clock actor the application must know in order to schedule resources that may have competing demands, maintenance requirements, and other outside issues that may affect a simulation. At a minimum, this object will be responsbile for providing start and stop times to manage simulations that may use a lot of unexpected resources. adiffer 2002/02/25
Sail Element A Sail Element is a description of a single atomic component of a solar sail that can be named, typed and desribed in terms of values that represent its extent adiffer 2002/02/25
Sail Model A Sail Model is a consistent set of Sail Elements with enough information to describe how those elements are constrained by each otherand external demands. A Sail Model is consistent when physical properties describing it have meaning like they would for a single atomic rigid body. adiffer 2002/02/25
Sail Property A Sail Property is a description of a single external property that can be named, typed and desribed in terms of values that represent its physical nature.

It is looking like this is most likely going to be an interface common to all the classes that may actually represent the details of the property.

adiffer 2002/06/06
Sail Property Set A Sail Property Set is a collection of Sail Properties with enough information to describe the complete external behavior of a Sail Model A Sail Property Set is complete when the physical properties within it have sufficient information within it to treat the Sail as a single, atomic rigid body in a simulation. adiffer 2002/02/23
Sail A Sail is a Sail Model and a Sail Property Set along with all the methods needed to encapsulate both of those objects from the users sufficiently and control the interactions.

The Sail must have some mechanism to flatten or serialize itself for export to files and transport over a network. No outside method is expected to know how to do this as well as the Sail itself will.

The old SailAway application has an existing Sail object that can be modified to meet the needs of the new design while retaining the already working parts of the physical model.

adiffer 2002/04/03
Command A Command is an order to a component of the application that results in changes and collects responses from the changed objects. Commands are the object representation of the Command API's of each package in the system. Each Command has a state and a sense of validity. adiffer 2002/02/25
Flight Assumption A Flight Assumption is some conditional statement made about a Sail component combined with a Command to be executed if the condition is met. The affected component could be a part of the Sail Model or one of the Sail Properties. Flight Assumptions are the conditional part of the Command language that allows adaptive bahavior within the System. At a minimum, a Flight Assumption has a state, a conditional statement, and a command. adiffer 2002/02/25
Flight Assumption Set A Flight Assumption Set is collection of Flight Assumptions attached to a particular Simulation. A Command Set is valid when it contains flight assumptions that are well-formed. adiffer 2002/02/25
Simulation A Simulation is a Sail, and Command Set, and an optional Flight Assumptions List. A Simulation is the highest object in the Physical Model and will be responsible for most of the interactions described in the PM API. Other packages will call here and the Simulation will pass down the orders and pass back the responses. adiffer 2002/04/03
Validation Test A Validation Test is some conditional statement made about an object that contains data provided by a user. Validation Tests are the screens that help the system determine whether to accept the associated object for use in a Simulation. Each Validation Test states what the Object must be like in order to pass.

Since a Validation Test is an object itself, it can also contain the results of the test. So, at a minimum, a Validation Test has a state, an object to be tested, a conditional statement, and a test result.

adiffer 2002/04/03
Validation Test List A Validation Test List is a collection of Validation Tests to be run upon a single Object.

A Validation Test List can handle the need to deliver a combined set of test results to any Object that calls for the validation tests in the first place. So, at a minimum, a Validation Test List has a state, a list of Validation Tests, and object to be tested, a test result summary, and enough methods to cycle through the Validation tests while keeping track of where it is and the first test to fail.

adiffer 2002/04/03
Receipt A Receipt is an partial echo of a Delivery Request with just enough information from the Library to tell a System that the remote request was received intact and that work is progressing or scheduled. adiffer 2002/04/03
Delivery Request A Delivery Request is a object that contains information needed by a Library to determine what is being asked for, who is asking for it, where it is to be delivered, and when it is to be delivered. Most of the time, the last two elements will default to the location of the calling object and immediately, but that may not always be true in the future. adiffer 2002/04/03
Status Log A Status Log is simply a collection of text lines written by various objects in the System that are permitted to inform the User of their activities. The Status Log will most likely get used by the GUI to allow a User to watch activity fly by or to debug a strange occurance later once it is output to an external file. adiffer 2002/04/03
Orbit An Orbit is one of the many possible Sail Properties. This particular one describes the physical details and laws involved with two-body gravitational orbits. adiffer 2002/04/03
Perturber A Perturber is one of the many possible Sail Properties. This particular one describes the physical details and laws associated with the celestial mechanics of small purturbations on two-body gravitational orbits. Different purturbation techniques will be represented by different Perturber Objects as the need to code them occurs. adiffer 2002/04/03
Force Model A Force Model is a classification of a certain group of possible Sail Properties. This group is needed by Perturbers to represent the actual perturbing forces experienced by a small body in a gravitational field caused by large masses. adiffer 2002/04/03
Radiation Force Model A Radiation Force Model is one type of Force Model that represents the laws of Radiation Pressure and how they affect the Sail Elements within a Sail Model.

One Radiation Force Model is used for each radiation source in the simulation. Usually, this source will be the sun, but it may be a planet bound laser instead if a researcher wishes to analyze laser driven sails.

adiffer 2002/04/03
Ephemerides Ephemerides is a catch-all object that will probably have to be defined in more detail as the system evolves. Objects in this category are ones the system may output as exported data meant to permit integration of the System with external applications commonly used in the Industry.

An example of this group is a collection of orbit positions and velocities at various times during a flight. A properly formatted file with this data could be generated by the System and exported for use by the Satellite Tool Kit from Analytical Graphics, Inc.

adiffer 2002/04/03
