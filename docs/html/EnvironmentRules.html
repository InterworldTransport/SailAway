SailAway Project
Environment Rules

Number Rule Type Static/Dynamic Rule Definition Source
# 1 Structural Fact Static A User may run a simulation in batch or interactive modes. Both options must be provided for all features. User Expectation
# 2 Action Triggering Dynamic A User with an internet connection that is not currently configured to make use of a library must be made aware of the option to store their simulated sails and flights in off-site libraries. Interworld Transport Management
# 3 Action Triggering Static A User entering the System must identify themselves enough to satisfy basic user validation screens. Anonymous identities will be available for those wishing to bypass certain processes at the cost of potential 'mistrust' by online application resources such as libraries and calculators. Interworld Transport Management
# 4 Action Triggering Dynamic A User with an anonymous identity must be identified as such to all off-site resources they may attempt to use. Interworld Transport Management
# 5 Action Restricting Dynamic All online resources made available to users as off-site resources must be able to deny service to anonymous users if they so wish. Interworld Transport Management
