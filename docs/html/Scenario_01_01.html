SailAway Project

Scenarios
Use Case Name Create a Sail Model
Iteration Finished
Scenario
  1. The User indicates they have information about a new sail element.
  2. The System assumes the new element is to be added to the current working sail model
  3. The System presents an interface for collecting sail element information.
  4. The User responds by entering new element information or cancelling the action.
  5. The System responds by passing the sail element to a validation process.
  6. The System responds by saving the new information as part of the current sail model if the element is judged to be valid.
  7. The System responds by informing the User whether the new element information was saved successfully or not.
  8. The new element infromation was not saved successfully, so the System proceeds silently.
