SailAway Project

Scenarios
Use Case Name Create Flight Simulation
Iteration Finished
Scenario
  1. The User indicates they have new information about an old flight simulation.
  2. The System presents an interface for collecting information about the old simulation so it may retrieve it.
  3. The User responds by providing information naming a simulation for retrieval or cancelling the action.
  4. The System responds by retrieving the old simulation.
  5. The System presents an interface for collecting information about the objects that make up the simulation and pre-populates it with information from the old simulation.
  6. The User responds by entering new object information or cancelling the action.
  7. The System responds by passing the changes to a validation process.
  8. The System responds by informing the User whether the changed simulation is well-formed or not.
  9. The System responds by saving the new information as part of the current flight simulation.
  10. The System responds by informing the User whether the new simulation information was saved successfully or not along with any relevant error information.
