Use Case Name Create Sail Property Set
Iteration Finished
Scenario
  1. The User indicates they have information about a new sail property.
  2. The System assumes the new property is to be added to the current working sail property set.
  3. The System presents an interface for collecting property information.
  4. The User responds by entering new property information or cancelling the action.
  5. The System responds by passing the property to a validation process.
  6. The System responds by saving the new information as part of the current sail property set if the property is judged to be valid.
  7. The System responds by informing the User whether the new property information was saved successfully or not and delivers all relevant error information.
