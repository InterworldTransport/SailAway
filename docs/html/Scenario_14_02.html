Use Case Name Create Sail Property Set
Iteration Finished
Scenario
  1. The User indicates they have new information about an old sail property.
  2. The System responds by presenting an interface to the user to determine which property is to be changed.
  3. The User responds by choosing an existing property from the working sail property set.
  4. The System presents an interface for collecting property information and pre-populates it with the old property information.
  5. The User responds by overwriting old information or entering new property information or cancelling the action.
  6. The System responds by passing the property to a validation process.
  7. The System responds by saving the new information as part of the current sail property set if the property is judged to be valid.
  8. The System responds by informing the User whether the new property information was saved successfully or not and delivers all relevant error information.
