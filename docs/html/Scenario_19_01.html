Use Case Name Iterate Sail One Phase Step
Iteration Finished
Scenario
  1. From a System UI, the user issues the command to step forward one phase step along the orbit using perturbations.
  2. The Sail uses a default phase step and issues the revised order to its Orbit as a discrete iteration of all related properties.
  3. The Orbit determines it is safe to use perturbation assumptions. The Orbit sets flags to signal downstream calculators.
  4. The Orbit then requests an update to the orbit properties from its Perturber as a small perturbation.
  5. The Perturber collects all the perturbing forces acting upon the Sail into a list.
  6. The Perturber takes the list and calculates resulting forces and torques from each of them.
  7. The Perturber takes the sum of the perturbing forces and calculates the changes to the orbit parameters.
  8. The Perturber takes the sum of the non-inertial perturbing forces and torques and sends a copy to the Sail.
  9. If the changes to the parameters do not pass the smallness test, the Perturber tries again with a phase step that is half the size of the previous value. Return to step 7.
  10. The Perturber hands the new orbit parameters to the Orbit.
  11. The Orbit derives the new location and velocity of the Sail based upon old orbit properties.
  12. The Orbit updates itself and passes control back to the Sail.
  13. The Sail updates itself and issues a statement signaling the completion of the order to the registered listeners.
