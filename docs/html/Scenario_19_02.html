Use Case Name Iterate Sail One Phase Step
Iteration Finished
Scenario
  1. From a System UI, the user issues the command to step forward one phase step along the orbit using perturbations.
  2. The Sail uses a default phase step and issues the revised order to its Orbit as a discrete iteration of all related properties.
  3. The Orbit determines it is not safe to use perturbation assumptions. The Orbit sets flags to signal downstream calculators.
  4. The Perturber collects all the perturbing forces acting upon the Sail into a list.
  5. The Perturber takes the list and calculates resulting forces and torques from each of them.
  6. The Perturber takes the sum of the non-inertial perturbing forces and torques and sends a copy to the Sail.
  7. The Orbit derives the new location and velocity of the Sail based upon old orbit properties.
  8. The Orbit updates itself but keeps its primary parameters and passes control back to the Sail.
  9. The Sail updates itself and issues a statement signaling the completion of the order to the registered listeners.
