Name Run a validation test
Iteration Finished
Summary In this case, all validators may reuse the structure of the test queue and the fabrication of the test results objects.
Basic Course of Events
  • The System accepts an object to be tested along with a test queue to be run.
  • The System creates a blank 'test result' and associates it with the element. Any old test results are deleted.
  • The System passes the object to be tested to a test in the test queue.
  • The test returns results to the System.
  • The results are appended to the test result.
  • If the test results are negative, processing stops and the test result is returned to the calling validator.
  • If the test results are positive and further tests need to be performed, processing returns to step 3 using the next queued test.
  • If the test results are positive and no further tests need to be perfomred, the test result is returned to the calling validator.

Extension Points .
Trigger Any object validator may call on this case.
Assumptions .
Preconditions
  • There must be an object to be tested
  • There must be a list of tests in a test queue.
  • The case calling this one must expect a test result in return
Postconditions A test result object will be returned to be used for determining whether the object being tested passed or not.
