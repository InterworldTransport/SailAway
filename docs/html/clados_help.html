Clados Package Overview

A Clifford Algebra is a mathematical object useful for representing geometry in an algebraic manner. The physics of celestial mechanics was born from geometry observations yet it is calculated today with algebra. Calculus has its roots in geometry and algebra. Uniting the two mathematical arenas gives those simulating the physics a very powerful programatic tool.

Clados = CLifford Algebra Derived ObjectS

There are other approaches to implementing geometry manipulation code. Most are based upon the handling of matricies. Using Clifford Algebras is simply better. Some knowledgeable people may debate this, but we don't. The reason we don't is that many programmatic errors are easily avoided with Clifford Algebra derived objects. Besides, the generators of Clifford Algebras can be cast as matrices with all the rules of matrix algebra reproducing the rules of geometric algebra. Using the matricies hides the underlying geometry and leads to unnecessary computation.

The clados package implements Clifford Algebra objects. These objects can represent any property from physics. The Moon's momentum and angular momentum can be written as a Monad. The inertia of an extended mass can be written as a dyad. >

The Monad

The Monad class is the basic single-algebra object generator in a flat space. Most of the physical properties taught in undergraduate physics can be represented with a single-algebra object.

The description of what a Monad is can be summarized as follows.

  • A name for the Monad
  • A name for the reference frame used to describe the flat tangent space. It is assumed a transformation on the reference frame will change this name to demonstrate the action.
  • A named location representing the foot of the tangent space. This location is the point on the underlying manifold where the tangent space is defined as tangent.
  • An array of coefficients from the field upon which the algebra is defined. Clados monads are defined upon real numbers.
  • The number of generators with a sqaure of positive unity.
  • The number of generators with a sqaure of negative unity.

The foot name is not used in any serious fashion at this time. Later versions of Clados will support transformations of the foot along the underlying manifold in some fashion, so it makes sense to include place holders in the monad now.

The frame name is used now as a way to ensure Monads using different reference frames may be treated as though they were of a different type without having to actually declare another class within the package. Passive transformations change the reference frame, but no support for them has been coded into the monad yet.

Several arrays are figured once the basic information has been provided in the list above. The generator and multiplication tables are both produced and kept with each instance of a Monad.

Not all possible metrics of a space are supported within the Monad. The generators with a positive square are listed in the generator table before the ones with negative square. No generality is lost with this limitation, though fans of certain metrics may find this annoying when they can't write a multivector the way they want. >

The Dyad

Some thought has been given concerning how to code a Dyad within Clados. There are some fundamental choices that have to be made early. We have yet to make them. How these choices work out will have a definite impact on how some physical properties like rotational inertia and the link between angular momentum and angular velocity are written.

A Dyad is the basic two-algebra object generator in a flat space. It may be thought of as two Monads written side-by-side with each using a different reference frame. It is possible for each monad to use the same foot location, but the reference frames are assumed to be different even if they appear to be interchangeable.

A Dyadic is a sum of dyads. All dyads may be written as a dyadic by breaking each Monad into a sum over its basis elements and them performing the multiplicative distribution. It is not so clear that all dyadics simplify to a product of Monads, however.

Geometry in three dimensional Euclidian spaces.

Newtonian physics is performed in a three dimensional space where time is treated as a parameter of our algebraic equations. Three dimensional spaces have four distinct ranks (sometimes called 'grades') for the geometric objects within them. The basic objects are as follows...

  • Points - grade 0
  • Line segments - grade 1
  • Plane segments - grade 2
  • Volume segments - grade 3
Each of these objects has associated with it a sense of magnitude. Each one except the points also has an associated orientation or direction.

Usage Example 1:
Imagine a room filled with air. Each location in the room has associated with it a temperature. The function representing the temperature anywhere takes as input a location defined by useful parameters and may output a point with a magnitude equal to the temperature there. 

f(location) = point*temperature
Most people don't bother writing down the 'point' in the algebraic equation, but it can be done. Don't confuse the parameters of the function describing the location with the point located there. Many do confuse the two, but they are not the same thing.

Usage Example 2:
Imagine the room from example one. If someone puts a fan in a corner and turns it on, the air will move around. Now each location within the room has associated with it a line segment representing the velocity of the air. 

f(location) = directed line segment*speed
Each line segment can be thought of as a line with an arrow describing which way it points. There are two directions a line segment could point, so we have a positive and negative possibility.

Usage Example 3:
With the same air filled room from the previous two examples, imagine that each molecule of air was also rotating. We can now associate a small plane segment with each location to represent the equators of each rotating molecule. 

f(location) = directed plane segment*rotation speed
Each plane segment can be thought of as a small loop with an arrow describing which way one goes around on it. Remember that a single plane at the equator can represent two possible rotations; east to west, and west to east. One direction can be positive while the other is negative. With this notion we turn plane segments into directed plane segments and grade two objects are avialble for use.

Many people are used to using a right hand rule at this point to turn a plane segment into a related vector perpendicular to the plane. That operation is called 'taking a dual' in a geometric algebra and is unecessary for our work. It is better to avoid using the dual for reasons that might become apparent when a 'multivector' is defined next.

All three examples can be combined and written as one function if the air in the room is warm, moving, and rotating. There is no reason why our function must stick to producing geometry of one grade. 

f(location) = 	point*temperature + 
		directed line segment*speed + 
		directed plane segment*rotation speed
It will be left to the reader to put the directed volume segment to work for the air filled room example. Think awhile about the Ideal gas law.

Algebra in three dimensional Euclidian spaces.

Many people find the geometric nature of Clifford Algebras relatively straight forward. Our intuitions seem well designed for geometry. Learning the algebraic nature is where most people bog down.

Newtonian physics is performed with equations and the other aspects of algebra. With its roots in geometry, celestial mechanics took a great leap forward once practitioners could operate algebraically. This section focuses upon how to write the geometry in a symbolic form that lends itself to easy use and understanding within the framework of algebra.

We start with how to write the various ranked objects in symbolic form.

  • A point is a number multiplied by the symbol 'e'
  • A directed line segment is a number multiplied by the symbol 'ei'
  • A directed plane segment is a number multiplied by the symbol 'eij'
  • A directed volume segment is a number multiplied by the symbol 'eijk'
  • where i, j, and k = 1, 2, or 3
  • and i not= j    and j not= k    and i not= k

There are special names given to each grade that you should probably learn if you ever wish to be able to read texts on the subject.

  • Points are refered to as scalars.
  • Directed line segments are refered to as vectors.
  • Directed plane segments are refered to as bivectors and sometimes as psuedovectors.
  • Directed volume segments are refered to as trivectors and often as psuedoscalars.

Now it's time to try a few examples of symbolic forms.

  • 5 is a number
  • 5e is a scalar
  • 0e3 is a vector
  • 3e1 is a vector
  • -5e2 is a vector
  • 0e12 is a bivector
  • 3e23 is a bivector
  • 5e123 is a trivector
  • 5e321 is a trivector

Now that we have defined the actors on our stage, we must define what they do. In algebra, this is done by defining what we mean by addition and multiplication. Don't assume these operators work the way they do for real numbers. They do in some ways and not in other ways.

The meaning of addition

The fastest way to describe how addition works is to give a few examples. Don't expect all the problems to simplify to a single symbol as they would with real numbers.

  • 3e + (-5e) = (-2e)
  • (-5e) + 3e = (-2e) {Yes...Addition is commutative}
  • 6e + 5e12 = 6e + 5e12 {No simplification}
  • (3e1 + 2e3) + (6e2 + -3e3) = (3e1 + 6e2 + -1e3)
Addition works by group terms with similar basis elements and adding the coefficients.

The meaning of multiplication

Multiplication is a trickier operation. It is best defined in terms of the basis vectors. The other possibilities fall out of this partial definition in a consistent manner.

  • ei * ej = eij if i not= j
  • ei * ej = e if i = j
  • and eij = -1eji
  • and (a * (b * c)) = ((a * b) * c)) for all a, b, c in the algebra

We can now derive how multiplication works for all other cases. Here are some examples.

  • 5e * 3e1 = 15e1
  • 3e3 * 2e12 = 6e312 = 6e123
  • -2e1 * (6e2 * 4e123) = +48e3
Note that multiplication is not always commutative.
Different basis vectors always anti-commute.

The multiplication table

Multiplying sums of geometry works by distribution.
(A+B)*(C+D)=A*B + A*D + B*C + B*D

Be careful to write the products in the correct order relative to multiplication.

Multivector

A result that contains a mixture of grades is called a 'multivector'.

Now that we have a working definition for addition and multiplication, we can apply the rest of the rules we learned from geometry. Be careful when trying to perform division or use higher level functions. There are many multivectors that have no multiplicative inverse besides 0e. For a little practice, see if you can find an object that acts as A-1 for the following multivectors.

  • A = e1
  • A = 0.5 * (1e + e1)
  • A = 0.5 * e2 * (1e + e1)
